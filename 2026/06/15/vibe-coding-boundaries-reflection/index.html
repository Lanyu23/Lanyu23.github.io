Vibe Coding 好用，但这次让我长了个心眼

一句话：Vibe Coding 确实好用，但如果权限和边界没说清楚，它也很容易听不懂人话，自己走到奇怪的方向上去。

最近用 Vibe Coding 写代码，整体感受还是很明显：它确实好用。

很多时候，我只要把需求大概说出来，它就能看项目、读代码、改文件、跑命令，把一堆原本很碎的操作串起来。以前自己写代码时，遇到小问题可能要查文档、翻文件、试命令，来回折腾半天；现在交给 AI，它经常能直接推到一个能看的状态。

但这次也算是给我上了一课。

好用是真的好用，容易走弯路也是真的容易走弯路。

这次的问题

我当时在用 Codex GPT-5.5 写代码，跟它说明代码里“生成桌面快捷方式”的功能有问题。

我的本意是让它去理解代码逻辑，看看为什么生成的桌面快捷方式不对，或者应该怎么改。结果它直接在我的本地电脑上测试了一次，真的生成了一个桌面快捷方式。

这个动作让我有点愣住。

因为在这之前，我从来没有明确让它在我的本地电脑上进行这种测试，更没有说“你可以在我桌面上生成文件”。我只是描述了代码里的问题，它却把这个问题理解成了“现在就在本机执行一次看看”。

这就很典型：AI 写代码很快，但它并不一定真的懂你话里的边界。

可能是谁的问题

这件事不能完全怪它，也不能完全怪我。

一种可能是我给了它完全访问权限。

当一个工具真的有能力读文件、改文件、跑命令，甚至影响本机环境时，它就不再只是一个聊天窗口。它已经变成了一个能动手的协作者。只要权限开得够大，它就可能把“我应该验证一下”理解成“我现在就可以在真实环境里验证一下”。

另一种可能是我一开始没有把边界说清楚。

比如我没有提前规定：不要在桌面、启动项、注册表、系统目录这些位置做真实写入；不要在没有确认的情况下创建快捷方式；测试必须放在临时目录；涉及本机环境变化之前必须先问我。

这些边界对我来说好像是默认常识，但对 AI 来说不一定。它看到的是任务、代码和权限，不一定能自动识别我心里那条“这里只是讨论代码，不是让你动我电脑”的线。

还有一种可能，就是最近 GPT-5.5 降智确实有点严重。

有些时候它好像没有通过场景语言识别出我想表达的问题。我说的是“代码里这个功能有问题”，它理解成了“你应该直接在本地跑一下这个功能”。这中间差了一层判断，但它偏偏跳过去了。

Vibe Coding 的风险

Vibe Coding 最舒服的地方，是你不用把每一步都想清楚。

你只要描述一个方向，它就能自己往前推进。这个体验很爽，也会让人越来越依赖它。问题是，同样因为它会自己往前推进，它也可能在你没注意的时候推进到不该推进的地方。

尤其是当它有完整文件权限和命令执行权限时，风险会被放大。

它可能会为了测试一个功能，真的改你的本机环境；可能会为了修一个 bug，引入一堆新逻辑；可能会为了绕过一个问题，走出一条看起来能跑、后面却很难维护的路。

修 bug 和后期维护困难，也是我这次更明显感受到的地方。

AI 生成代码的时候，经常能很快把问题“糊住”。但如果一开始方向偏了，后面再修就会越来越麻烦。你以为它是在帮你补一个洞，结果它可能是在旁边又挖了两个坑，只是暂时没塌。

我以后会怎么用

这次之后，我大概会给自己定几个规则。

第一，涉及真实本机环境的操作，要提前说清楚边界。

比如桌面、系统目录、启动项、注册表、环境变量、账号配置这些地方，默认都不能直接动。要动可以，先说明原因、影响范围和回滚方式，再让我确认。

第二，测试优先放在临时目录或项目内部。

如果只是验证“生成快捷方式”的代码逻辑，完全可以先在临时目录里造一个假路径，或者只检查生成参数和目标内容，不需要直接在真实桌面上试。

第三，不要让 AI 一路自己脑补到底。

Vibe Coding 很容易让人变懒，觉得“它应该懂”。但现在看来，关键边界还是要讲清楚。尤其是权限开得越大，越不能只靠默契。

第四，生成代码之后还是要看结构。

能跑不代表好维护。AI 写出来的东西，如果抽象很乱、路径写死、异常处理随便、测试逻辑混在业务逻辑里，后面修起来会很痛苦。不能只看它当下有没有把报错消掉。

现在的感觉

我还是觉得 Vibe Coding 好用。

它能把很多重复、琐碎、没意思的编码工作变快，也确实能让我更愿意开始做事。这点没有变。

但我现在也更清楚，它不是一个天然懂边界的工具。它有能力做很多事，不代表它每次都知道哪些事应该先问。尤其是当它拿到了完全访问权限之后，它的“主动性”就会从优点变成风险。

所以这次算是长个心眼。

以后再用这类工具，我会更早把边界写清楚：哪些地方不能动，哪些操作必须确认，哪些测试只能在临时环境里做。AI 可以帮我写代码，可以帮我排查问题，但不能默认它可以替我决定本机环境该怎么被改。

Vibe Coding 不是不能用，而是越好用，越要会用。

工具越强，越不能把方向盘完全松开。
